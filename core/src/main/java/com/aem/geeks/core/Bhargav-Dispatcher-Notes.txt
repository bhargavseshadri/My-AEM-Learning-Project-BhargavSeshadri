--> Important points
- Dispatcher does not perform tasks, dispatcher is a module in our apache web server that tells Apache webserver what tasks need to perform.
- Caching, load balancing, access and security all these are the feature of APACHE webserver, but the dispatcher tell the server
  to create the cache as per the provided configurations.
- Using rewrite rules we will redirect the pages :: Eg: if we hit www.intel.com  -> it will redirects to -> https://www.intel.com/content/www/us/en/homepage.html

---> Difference between cache deletion, cache invalidation, cache purge
- Cache Deletion : Physically removing cached files from filesystem. so the next request goes to publisher and then page served and a new cache file got created.

- Cache Invalidation : Marking cache as stale or unusable without deleting file immediately.
                        - In this dipatcher checks the ".stat timestamp > cached file timestamp", then cached file considered outdated
                        - dispatcher fetches fresh content and overwrites cache
                        - Old file exists physically but is logically invalid.

- Cache Purge : “Cache purge” is most commonly associated with CDNs.
                 - it usually means Explicit forced removal of cached content from caching layer like cdn.

- cache flush : It basically means cache invalidation happened.

---> Sections in Dispatcher.any
- /clientheaders : tells what headers are allowed through request.
- /virtualhosts or vhose : - it define which domain is mapped or resolved to which content. using this we can have multiple domains with multiple configuration.
                  - we can have multiple /virtualhosts.
                   Eg : www.intel.com --> maps to AEM Sites
                        www.intelDigitalLibrary.com ---> maps to AEM Asset.
- /renders : using this we tell the server from where it can get and serve the response for the incomming request.
- /filter : when we want to filter the request we use this. basically we dont want to send all the requests to our AEM server, we only send some specific
            filtered requests. those specifications will be done here.
- /cache : The cache section regulates what responses will be cached and where.
- /invalidate :
- /allowedClients : This tell to the server which are the clients(AEM Author or AEM publisher etc) are allowed to clear the cache.
